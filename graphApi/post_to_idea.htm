Posts to Idea Entities

Post requests are handled as defined here when the target is an entity of type Idea.  The appropriate response is determined by the action and type post properties.

Post a comment

Post property

Datatype

Req

Description

action

String

MAY

"post" or undefined

type

String

MUST

"Comment"

content

String

MUST

The text of the comment.

Returns:  The created Comment entity.

Post a new comment to the idea's comment thread, in other words the idea's discussion.

 

Post a vote

Post property

Datatype

Req

Description

action

String

MAY

"post" or undefined

type

String

MUST

"Vote"

value

String

MUST

Appropriate value depends on the configured voting style for this idea's inquiry.  That value may be retrieved by looking at:

 thisIdea.inquiry.ballot.type

(Get the inquiry for this idea, the preferred ballot for that inquiry, and the type of that ballot.)

If ballot type is UpDownBallot:

value may be: "up", "down", -1, 1, 0

If ballot type is StarBallot:

value may be a number in the star range, usually 1 to 5

Returns:  The created Vote object.

Post a vote against the comment.  Votes can only be posted for the current user, in other words the user represented by the current access token.  It is not possible to post a vote on behalf of someone else.

The expected value depends on the ballot being used by the idea's inquiry.  If it is an UpDownBallot, the value may be "up" or 1 to register a "thumbs up" vote,  "down" or -1 both register a "thumbs down" vote, or 0 clears the previous vote.  If it is a StarBallot, the value should be a number in the star range, in other words between 1 and 5.

 

Post a prediction review

Post property

Datatype

Req

Description

action

String

MAY

"post" or undefined

type

String

MUST

"Review"

benchmark

Questionnaire

MUST

This SHOULD be the questionnaire object value that was returned by an earlier  "getPredictionQuestionnaire" post query to the idea's inquiry.  It is critical that the basis property of that questionnaire is included in the questionnaire value given as the benchmark here.  

values

PlainObject

MUST

The properties of this plain object correspond to the fields in the questionnaire's form.  For prediction reviews, these fields will all have a promptStyle value of "higherLower".  This means that the properties in this values plain object MUST all have one of the two text values:

"higher" or "lower"

These values are understood to be relative to the corresponding values in the questionnaire's basis.

Returns:  The created Review object.

This posts a prediction review to an idea in response to a prediction questionnaire that was retrieved by an earlier "getPredictionQuestionnaire" post query to the idea's inquiry.
