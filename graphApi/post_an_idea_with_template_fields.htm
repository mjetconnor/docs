Post an Idea with Template Fields

Many SpigitEngage communities and challenges are configured to have an "idea template".  The idea template defines additional properties that an idea might have.   Some of those properties are required, which means that an idea posting that does not include them will be rejected.  In this regard, the earlier tutorial page that described how to post ideas was a little to simple.

The first step is to get the object that describes the idea template that is in effect for a particular inquiry.  Every inquiry has a governor property, whose value is an entity of type Governor.  For inquiries where an idea template has been defined, that governor will have a uiForm property, whose value is a Form.

You can get that form directly in a query like this (still using "_ds:q-1" as our example inquiry).  Using the pageSize query parameter with a large value insures that the response will include all of the fields in the idea template form.

GET https://www.example.mindjet.com/api/ds1/q-1/governor/uiForm?pageSize=100

Here is a response example:

Each of the fields in the form correspond to an idea template property.  In the above example, there are three fields.  The name property of each field indicates the name of a property in an idea entity that is managed by this template.  In this case they are Elevator, Description and Innovation.

Notice the required property in each field.  Elevator and Description are required, Innovation is not.  In the context of posting new ideas to this inquiry, this means that posts MUST include values for the Elevator and Description properties, and MAY include a value for the Innovation property.  Every idea requires a title property regardless of the template.

POST https://www.example.mindjet.com/api/ds1/q-1

with headers:

Content-Type: application/json

Authorization: Bearer 914a9b4d83fa7333d3b517643a30bac2

and request body:

{

   type: "Idea",

   title: "Buy low, sell high",

   Description: "If you do this, you get the biggest margins",

   Elevator: "Quick way to make lots of money",

   Innovation: "Don't buy high"

}

The fieldType property in each field indicates what kind of value is expected for each property.
