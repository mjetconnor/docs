Error Message

If an error occurs while processing any of these requests, the response may be an error message.  Depending on the circumstances of the query and error, these values may be returned either as the body of a response or as query parameters appended to a redirect URI.

Response Element

Description

error

One of a set of enumerated code values, listed below.

error_description

Additional explanatory text about the error.

state

If this error is in response to a request where the state query parameter was specified, its value is echoed back here.

The appropriate HTTP status code will also be set on the response.  It will most likely be one of these:

400                Bad Request

401                Unauthorized


Values that may be included in the error element:

Error Code

Description

invalid_request

The request was invalid in some way.  If possible, one of the other more specific error codes will be used.

invalid_client

The specified client_id does not identify a recognized application.

invalid_grant

 

invalid_protocol

The given TCP protocol is not allowed.  This most likely means that a protocol other than HTTPS was used.

unauthorized_client

 

redirect_uri_mismatch

The redirect_uri value given does not match any allowed pattern specified by the registered application.

access_denied

An invalid or lapsed access token was given, or the access token does not provide sufficient permission for the request.

unsupported_response_type

An unsupported value was given for the response_type parameter.

unsupported_grant_type

An unsupported value was given for the grant_type parameter.

unsupported_content_type

An unsupported MIME type was used in the Content-Type request header.
