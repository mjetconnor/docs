Post an Idea with a Category

When a SpigitEngage inquiry (a community or a challenge) has been configured to have categories, one of those categories must be specified when posting ideas to it.  Get the categories for an inquiry by looking at its concepts property.  

GET https://www.example.mindjet.com/api/ds1/q-1/concepts?fields=title

which returns something like this:

{

  "version": "1.0",

  "query": "GET http://www.example.mindjet.com/api/ds1/q-1/concepts",

  "xmlns": {

    "sief": "http://purl.org/sief/2013/10/",

    "_ds": "http://www.example.mindjet.com/api/ds1/",

    "_host": "http://www.example.mindjet.com/api/"

  },

  "results": [

    {

      "data": [

        {

          "eid": "_ds:node-20",

          "type": "Concept",

          "title": "Technology"

        },

        {

          "eid": "_ds:node-22",

          "type": "Concept",

          "title": "Biology"

        },

        {

          "eid": "_ds:node-39",

          "type": "Concept",

          "title": "Marketing"

        },

      ]

    }

  ]

}

 

This inquiry has three categories defined.  As you can see, each of them are represented in the MindjetGraph data model as Concept entities.

To post an idea to this inquiry, we must pick one of these and specify it as the value of the category property in the POST message:

POST https://www.example.mindjet.com/api/ds1/q-1

with the usual headers, and request body:

{

   type: "Idea",

   title: "Buy low, sell high",

   category: "http://www.example.mindjet.com/api/ds1/node-39",

   Description: "If you do this, you get the biggest margins",

   Elevator: "Quick way to make lots of money",

   Innovation: "Don't buy high"

}
