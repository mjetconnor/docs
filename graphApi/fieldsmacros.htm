Field Macros

There may be a need to repeat complex field lists in a query.  For instance, you may want a certain set of properties to be returned for user entities, and there may be several different properties in a response that have user values.  It would be helpful to not have to repeat that list in the fields query parameter.  We have macros for that.

A macro name is any name with an "@" at the beginning.  A macro definition correlates a macro name with a field list.  The macro name can then be used as a reference to that list in place of the full value.

Example:

fields=

title,

description,

@userProps=[name,givenName,mbox,gender],

creator:@userProps,

inquiries:[creator:@userProps,title]*20

In this example the @userProps macro is defined and then used in two places.  This fields value is identical to this expanded variation:

fields=

title,

description,

creator:[name,givenName,mbox,gender],

inquiries:[creator:[name,givenName,mbox,gender],title]*20

To be sure, the second case of this example is no longer than the first.  It is actually a little bit shorter.  But it does help to make clear that the exact same inclusion rule is used for both creator properties.  As field lists get more complex, this feature becomes all the more useful.

Macro definitions may appear at any position in the list.  In particular, they do not have to appear before their first reference.  For readability, you may choose to group of all your macro definitions at the beginning or end of the fields list, or anywhere in between:

fields=

title,

description,

creator:@userProps,

inquiries:[creator:@userProps,title]*20,

@userProps=[name,givenName,mbox,gender]


fields=

@userProps=[name,givenName,mbox,gender],

title,

description,

creator:@userProps,

inquiries:[creator:@userProps,title]*20

Here is a more involved example that uses some nested PEs as well:

Example:

fields=

title,

description,

creator:@userProps,

inquiries:@inquiryProps*20,

@userProps=[name,givenName,mbox,gender],

@inquiryProps=[creator:@userProps,title,ideaSet:[ideas:@ideaProps*3]],

@ideaProps=[creator:@userProps,title]

The equivalent fully expanded variation is:

fields=

title,

description,

creator:[name,givenName,mbox,gender],

inquiries:[

creator:[name,givenName,mbox,gender],

title,

ideaSet:[

ideas:[

creator:[name,givenName,mbox,gender],

title

]*3

]

]*20

Use whichever form works best for you.


Scope of nested macro definitions

Macro definitions are a kind of field phrase, just like property expressions.  They may appear anywhere that a property expression might appear.  In particular, they do not have to be elements in the top level fields list but may be elements in a nested field list.  When that is the case, the scope of the macro definition is limited to the fields list that it appears in.

Example:

fields=

title,

inquiries:[

@user=[name,description],

creator:@user,

ideaSet:[

@user=[name,givenName,mbox],

ideas:[creator:@user,title]

]

]
