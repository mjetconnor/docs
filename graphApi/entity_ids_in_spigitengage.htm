Entity IDs in SpigitEngage

All MindjetGraph entity IDs (see Entity Concepts or Entity Class) begin with a gatewayURI.  For SpigitEngage instances, that is always the URL of the primary community there, plus the path segment "/api".  The primary community is often (but not always!) the one that starts with "www.".  For example, if the primary community has this URL:

https://www.mycompany.spigit.com

or

https://main.xyzcorp.spigit.com

then the gatewayURI part of entity IDs in this instance will be:

https://www.mycompany.spigit.com/api

or

https://main.xyzcorp.spigit.com/api

Every SpigitEngage instance has exactly one dataSpace entity and one tenant entity.  (See Community and Tenant Concepts.)  These entities always have the same entity keys:

Key for the dataspace

ds1

Key for the tenant

tenant-1

Since the dataSpace key is always "ds1", we can work out the common namespace prefixes used for REST queries this way:

_host:

primaryCommunityURI/api

https://www.mycompany.spigit.com/api
https://main.xyzcorp.spigit.com/api

_ds:

primaryCommunityURI/api/ds1

https://www.mycompany.spigit.com/api/ds1
https://main.xyzcorp.spigit.com/api/ds1

which makes the entity IDs for the dataSpace and tenant entities to be the following, taking advantage of the "_ds:" prefix:

dataspace

_ds:ds1

https://www.mycompany.spigit.com/api/ds1/ds1
https://main.xyzcorp.spigit.com/api/ds1/ds1

tenant

_ds:tenant-1

https://www.mycompany.spigit.com/api/ds1/tenant-1
https://main.xyzcorp.spigit.com/api/ds1/tenant-1

All of this information is provided mostly for your understanding in recognizing values and manual navigation.  It is very unlikely that you will ever need these elements when coding an application.  You should always get the initial entity IDs from the response to the access token request.  And you should always use the entity IDs and prefixes returned in subsequent queries to form new requests (i.e.. "walk the graph").  You should never assume, outside of a development or test environment, that you know in advance what these values are.
