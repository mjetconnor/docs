Registered Applications

All REST API calls are processed in the context of a registered application.  Using the OAuth 2.0 protocol, an access token is granted to a registered application executing on behalf of an authenticated user

Applications may be registered by admins for your SpigitEngage instance in the page: Administration Settings > API Management.

1.   Login to your SpigitEngage instance with an account that has administrative privileges.

2.   Navigate to the Administration Page.

3.   Click the API Management Icon.          

4.   Fill in the details for your application.

5.   Click Add App to submit your application.

 

 

Field

Description

Title

Title of the Application

Description

Describe the nature of the application and what it features in terms of SpigitEngage functionality

Status

Determines if the Application is Enabled or Disabled, If Status = Off the Application will be denied access to the API

Require User Authentication

Will prompt a user who accesses the application for the first time to authorize application to perform functions in SpigitEngage on their behalf.  (If On, user will be prompted, if Off, user will not be prompted)

Callback URL

When an application requests a token in order to gain access to the API, this URL will be used to redirect the response.  The response will include the token as well as other valuable information to assist the Application in determining the context of the user and site.

App Image URL

Used in the prompt for user Authorization, will display the image as part of the prompt.

Contact Name, Email, Phone

Who should be contacted if our Operations team notices an issue with the application that may require that the app be disabled or to troubleshoot issues with the application?

 

When the Application is registered, it will be assigned an App ID and an App Secret.  These are required in order to request a token to gain access to the API.  Please make note of these for future reference.

Authorization

The MindjetGraph REST API uses the OAuth 2.0 protocol to generate access tokens and use access tokens to authenticate REST queries.  The details of that process is described in detail here.  Those pages refer to an authorizationServiceURL.

For SpigitEngage, use the URL of any SpigitEngage community as the authorizationServiceURL for making these OAuth calls.  That community provides the context and rules for logging a user in, assuming there is not already an active SpigitEngage user session in the application's environment, whatever that may be.  It is not uncommon for different SpigitEngage communities, even within the same installation, to have differing login authorities such as different SSO or LDAP connections.  

Access tokens time out after 10 minutes of no use.  Not after an absolute elapsed time of 10 minutes.  This is admin configurable (somehow).
