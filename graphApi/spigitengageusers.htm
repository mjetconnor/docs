SpigitEngage Users

SpigitEngage users are represented as User entities in MindjetGraph.  A given user has the same entity object throughout the tenant (aka instance), irrespective of community.

In the Mindjet Graph model, User is a sub-class of Person, which is a sub-class of Agent.  Whereever you see entity properties defined to be agents, you will almost sure find a user entity there.

The final response from the OAuth authorization flows - the Access Token Response - includes the entity ID of the user that was just authenticated.  For SpigitEngage, this will always refer to a User entity.

In SpigitEngage, there is a one-to-one relationship between User and UserAccount entities.  The UserAccount entity is where the user's "username" is found.

SpigitEngage may capture additional information about a user when an external identity provider such as SAML or LDAP is used.  This extended information is referred to as user attributes.  These attributes may be used as the basis for limiting membership to communities that are configured as "private -- user attribute" communities.

These attribute values are available as properties in a user's entity object.  They must be explicitly requested by name however, using some variation of the REST API's fields query parameter.  User attributes are not included when using the default form that retrieves all properties for a particular user entity.
