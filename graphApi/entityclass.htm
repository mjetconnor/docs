Entity Class

Concepts: Entity Concepts

Everything that is persisted in the Mindjet Platform is an entity.  Put another way, anything that has a persistent ID, and can be retrieved by that ID at a later time, is an entity.

Mindjet entities are a kind of RDF resource.  In essence, entities are how the Mindjet Platform represents RDF resources.  The Mindjet Platform can (in theory - coming soon-ish!) handle resources that are served by applications external to Mindjet.  In these specifications, those are referred to by the more general class “rdf:Resource”.

The URI identifier for the entity is the value of the "eid" property.  The form of this URI is:

gatewayUrl / dataSpaceKey / entityKey

Mindjet Entities always have a set of basic managerial properties such as “creator”, “created” and “status”.  This is not a requirement of RDF resources.  In fact it is perfectly legal for a resource to have no properties at all, or just one or two simple properties.  But when a resource is a Mindjet Entity, we commit to it having the basic set of properties defined below.
