Get Data for an Entity

Every REST query in the MindjetGraph API is authenticated by specifying your access token in the "Authorization" HTTP header.  The value of that header must be the literal text "Bearer " (the word "Bearer" followed by a single blank space character) followed by your access token.  For example:

Authorization

Bearer 914a9b4d83fa7333d3b517643a30bac2

The authentication response from the previous tutorial step gave us two starting points for the exploration of your MindjetGraph data.  The userId value is the "entity ID" of the user that was just authenticated.  The communityId value is the entity ID of the community that was the basis for the authentication.

The value for communityId from the previous step was "https://www.example.mindjet.com/api/ds1/comm-1".

To get the data for an entity, any MindjetGraph entity, simply execute an HTTP GET request using the entity's ID as the URL of the query.

GET https://www.example.mindjet.com/api/ds1/comm-1

Don't forget the "Authorization" HTTP header.  Otherwise the request will be rejected with an access-denied error.

The response is in JSON format.  The following is a (somewhat abbreviated) example of what you might expect to get.  It starts with some managerial information -- the response version, a repeat of the query URL, and relevant namespace bindings.  Then there is the data of the requested entity.

Let's look a little closer at this entity's first two properties, eid and type.  The eid value is the entity's ID, which we are happy to see matches what we asked for.  Well, almost.  The value in the response uses the XML namespace prefix "_ds", whose mapped value is given earlier in the response.  If you replace the "_ds:" prefix with the its corresponding value, you get exactly the URI we used to make the query in the first place: "_ds" + "comm-1" = "https://www.example.mindjet.com/api/ds1/comm-1".

The type value is "Community".  This tells us that the datatype for the entity is the MindjetGraph class Community.  Look up that class in the Reference section to find out the details about what properties may be expected for this kind of object.

It is really important to note that the key value in the entity ID, in this case "comm-1", does not tell you what kind of object this is, even though it may look like it sometimes.  To know the type of a particular MindjetGraph entity, you must get its data and look at its type property.  It cannot be inferred from the entity's ID.
