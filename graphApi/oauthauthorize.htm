/oauth/authorize

Execute an HTTP GET request to this URL:

authorizationServiceURL/oauth/authorize?queryParam&...


Query Parameter

Required

Description

client_id

yes

The key of the application that is making this call.

redirect_uri

yes

The response to this authentication request will be sent by way of a 302 redirect to this URI.  The request will be rejected if this redirect URI is not compatible with the application's registration information.

response_type

yes

Must be either "code" or "token".  See below for details.

scope

no

Currently not supported.

state

no

A value that is echoed back in the response without modification.


Authorization requests sent to this URI always respond by way of 302 redirects.  Nothing is returned directly in the responses to these queries, other than error messages such as a missing or rejected redirect URI.

 

response_type

Description of Response

response_type=code

The redirected URL is extended to include an authorization code to be used in a subsequent request to a /oauth/token URI.  The redirected URI will have this format:

redirectUri?code=authorizationCode [ &state=stateValue ]

response_type=token

The redirected URL is extended to include query parameters that represent an access token response.  The redirected URI will have this format:

redirectUri#access_token=accessToken&...other response parameters

Note that the response parameters are included after a hash tag ("#").  This insures that these values do not travel any further than the browser or user agent that is receiving the 302 response.
