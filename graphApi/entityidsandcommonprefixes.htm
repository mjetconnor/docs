Entity IDs and common prefixes

As discussed in Entity Concepts, entity IDs are URIs that always have this format:

gatewayURL / dataspaceKey / entityKey

Example:

http://xyz.mindjet.com/graph/nyc/u512-312.b

has these components:

gateway URL       http://xyz.mindjet.com/graph
dataspace key     nyc
entity key            u512-312.b

When entity IDs are returned as part of a response, they are generally shortened by using XML namespace prefixes.

The prefix "_host" usually refers to the gatewayURI used in the query.

The prefix "_ds" usually refers to the combination of gatewayURI and dataspace key from the query, in other words everything except the entity key.

Sticking with the example above, these prefixes would be mapped this way:

_host: http://mindjet.mindjet.com/graph
_ds: http://mindjet.mindjet.com/graph/nyc

So for example, the entity ID "http://xyz.mindjet.com/graph/nyc/user-5123" might be represented instead as either "_ds:user-5123" or " _host:nyc/user-5123".

The exact mappings for these prefixes is always given in each response in the top level "xmlns" element.
