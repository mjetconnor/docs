Query Parameters

 

All MindjetGraph entities have identifiers that are URIs.  The simplest way to get the data for an entity is to execute an HTTP GET request directly on that URI.  The only missing piece is to be sure to include your OAuth 2.0 access token in the "Authorization" header of the query.

Remember that when you got an access token from an OAuth 2.0 request, the response also included the entity IDs for the authenticated user and home community.  These IDs are excellent URIs to use as starting points to navigate the graph.

HTTP Headers

Header

Required

Description

Authorization

yes

Value MUST be "Bearer: <your-access-token-here>"

Content-Type

no

Specifies the desired response format.  If not specified, the default is "application/json".

The response can be in various formats, as requested by the Content-Type header.  Examples in these Help Docs will generally use JSON.

See Response Format for a complete description of the structure of responses.  Regardless of content type, kind of query or HTTP method, all responses have a common structure.

GET https://www.example.mindjet.com/api/ds1/tenant-1
