Property Segments

The query URL may be extended by adding a path segment made up of a slash ("/") and a property name.  You can think of this as being like a dot operator in a script language.  The query's result will be the value of the property as derived from the identified root entity.

entityURI/propertyName

(Remember that entity URIs always look like this: gatewayURI/dataspaceKey/entityKey.)


Example:

GET https://www.example.mindjet.com/api/ds1/tenant-1/homeCommunity

where

entity URI https://www.example.mindjet.com/api/ds1/tenant-1
property name homeCommunity


The result of this query will be not the value for the tenant entity but rather the value of the homeCommunity property of the tenant entity.

This construct may be cascaded by tagging another property name segment onto the query URL, again in the same manner as a series of dot operators.

entityURI/propertyName/propertyName/...

Example:

GET https://www.example.mindjet.com/api/ds1/tenant-1/homeCommunity/title

where

entity URI https://www.example.mindjet.com/api/ds1/tenant-1
property name homeCommunity
property name title

The result of this query will be the title of the home community of the tenant entity.

Example:

GET https://www.example.mindjet.com/api/ds1/tenant-1/homeCommunity/img/url

where

entity URI https://www.example.mindjet.com/api/ds1/tenant-1
property name homeCommunity
property name img
property name url

The result of this query will be the url of the logo of the home community of the tenant entity.  Note by the way that images are usually not entities.  This example shows that queries with property selection is not limited to just entities, although of course the initial URI must be for an entity.
