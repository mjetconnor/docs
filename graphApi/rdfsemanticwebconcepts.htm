RDF (Semantic Web Concepts)

MindjetGraph is based on the standard data model of the Semantic Web, the Resource Description Framework (RDF).  RDF is the Internet's data architecture.  It has been a W3C (World Wide Web Consortium) recommendation since 1999.

A Resource is any kind of identifiable thing.  Because this is the Internet data model, those identities are (surprise!) nothing other than URIs.  You can think of a resource as being nothing other than its URI, together with all the things that relate to it.

Statements define those relationships. Every statements has three parts:

[ subject, predicate, object ]

or similarly:

[ subject, property, object ]

Subjects are always resources, in other words URIs.  Properties are also identified by URIs.  Objects may be resources (URIs) or they may be literal values such as text strings, numbers or dates.   For the obvious reason, statements are often called triples.

XML namespace prefixes are often used to represent those bulky URIs more concisely.  For example, RDF's own vocabulary is defined in the "http://www.w3.org/1999/02/22-rdf-syntax-ns#" namespace, typically using the prefix "rdf:".  So instead of referring to the property whose full URI is "http://www.w3.org/1999/02/22-rdf-syntax-ns#type", we usually talk about "rdf:type" instead.

"rdf:type" is an interesting property to take note of.  In RDF, the type of a resource is indicated by its "rdf:type" property, or more precisely by a statement where the subject is the resource's URI, the property is "rdf:type", and the object is the URI of a datatype.  In other words, the type of a resource is a property of the resource.

A group of statements taken together are called a graph.  A picture of an RDF graph typically looks like a bunch of dots that represent resources or literals, with arrows between them that represent properties.  Note that in an RDF graph those arrows have a direction, pointing from subject to object.

RDF has several commonly recognized serialization formats.  The W3C has declared an XML standard known as RDF-XML.  Other frequently used representations are Turtle and N-Triples.  There is also a proposed JSON standard format.

The vocabulary defined in "rdf:" provides just the barest basics to represent a graph of resources.  The sister vocabulary "rdfs:" (RDF Schema) adds definitions for "Class" and "Resource", and very basic notions such as "subClassOf."  The Web Ontology Language (OWL), typically prefixed as "owl:", provides a much richer way of adding structure to a graph, such as declaring that two properties are the inverse of each other.
