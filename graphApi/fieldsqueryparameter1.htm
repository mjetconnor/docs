Fields Query Parameter

The value of the fields query parameter is a field list, which is a comma-delimited list of field phrases.  The most common field phrase is a property expression.  The simplest property expression is just the name of a property.  So, the simplest field list is just a comma-delimited list of property names.

Examples:

...?fields=title,creator,created

...?fields=name,dateOfBirth,gender,mbox

For the first example, the response will include the "title", "creator" and "created" properties of the result object.  If the result is an entity, the "eid" and "type" properties are also included because they always are included for entities.  No other properties are included for the top level result object.

This list only applies to the top level entity in the response.  For nested entities, the default rule still applies, which is the summary rule.  In the first example, assuming that the value of the "creator" property is a User entity, the properties included in the response for that entity will be the user's "eid", "type" (presumably "User") and "name" if they have one.

Property expressions, described in the next topic, are used to instruct how to return nested values.

Whitespace

The fields list syntax is happy with whitespace.  Some of the examples in this document will show fields values using newlines and tabs to help illustrate the structure of complex expressions.  Remember though that URL encoding is not so accepting of whitespace.  When a field list is specified using the fields query parameter, be sure to URL encode as necessary ("+" or "%20" for blank space, for instance).  It may be easiest to just trim the whitespace out for actual queries.
