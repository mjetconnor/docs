DataSpace Class

Concepts: Community and Tenant Concepts

Every entity in the Mindjet platform, which is to say every persisted thing, belongs to a dataSpace.  The entity’s ID is allocated within the context of that dataSpace.  The dataSpace’s own entityKey is also included in entity’s expanded identity URL.   The simplest form of access control happens at this level as well.  

There is a close relationship between DataSpaces, Tenants and Communities, but those are all different elements that serve slightly different purposes.  Every dataSpace relates to (“belongs to”) a single tenant.  Most tenants relate to a single dataSpace, so that there is often a simple direct one-to-one relationship between tenants and dataspaces.  It is possible though for a tenant to have multiple dataspaces related to it, although every tenant must have at least one.

DataSpaces are themselves Entities, and so belong to a dataspace.  The dataspace that contains dataspaces is knows as a “gateway” dataspace.  The gateway dataspace is identified by the host part of a full entity ID URI.  The dataspace Key in a full entity ID URI then identifies a dataspace from within that gateway.  Gateway dataspaces contain themselves.  The “dataSpace” property of a gateway dataspace is itself.
