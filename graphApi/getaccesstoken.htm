Get an Access Token

Before you can do anything else, you have to get an access token.  Every REST query to the MindjetGraph API needs an access token in order to authenticate the query.  MindjetGraph uses the OAuth 2.0 protocol to get it to you.  

To try out the queries in this tutorial for yourself (please do!), you will need to have your own login to a SpigitEngage community somewhere.  We use the "www.example.mindjet.com" community in these examples.  Replace that domain name with the one for your own SpigitEngage community.

Let's get going by using the following query to get an access token.  Simply cut-and-paste it into your browser address bar, being sure to replace "www.example.mindjet.com" with your community's domain name.

https://www.example.mindjet.com/oauth/authorize?response_type=token&client_id=BeX5vrNVW7vq&redirect_uri=https://www.example.mindjet.com/html/api/testOauthRedirectPage.html

This is just one line.  Ignore any text wrapping that might appear in the display.

This is an example of using  OAuth 2's User Agent Flow.   A request is sent to the "/oauth/authorize" endpoint for your SpigitEngage community.  It has three query parameters:

response_type

token

Identifies the OAuth 2 flow

client_id

BeX5vrNVW7vq

The ID for the universal  tutorial application

redirect_uri

yourCommunityURL /html/api/testOauthRedirectPage.html

The access token is sent here by way of a 302 redirection

>> To make your own application ID (aka "client_id"), see SpigitEngage in the Graph :: Registered Applications.

If you are not already logged into that SpigitEngage instance, you will be redirected to the login page appropriate for that community.  The normal SpigitEngage login sequence will apply in accordance with how that community is set up.

Then, if you have never authenticated using this app before, which will surely be the case the first time you try this, you will be redirected to a page that asks if it is OK for this app, namely your tutorial queries, to execute under your name and with your permission.

After those hurdles are cleared, your browser is finally redirected to the "redirect_uri" that we specified in the original request.  The access token, and some other useful info, is appended as the hash tag part of that URI (reference details here).  

"/html/api/testOauthRedirectPage.html" is a page that we include in your SpigitEngage instance just to make it easy for you to manually try out OAuth 2 requests.  This page simply grabs the values that come to it in the hashtag part of the request and displays them in a way that is easy for you to see.  Your app will do something else.  This information is just for your app.  End-users really don't want to see this stuff.

The final redirected URL will look something like this:

https://www.example.mindjet.com/html/api/testOauthRedirectPage.html#issued_at=1385422189287&userId=https%3A%2F%2Fwww.example.mindjet.com%2Fapi%2Fds1%2Fuser-523&refresh_token=basIJZuxSdzh&access_token=914a9b4d83fa7333d3b517643a30bac2&&communityId=https%3A%2F%2Fwww.example.mindjet.com%2Fapi%2Fds1%2Fcomm-1

and "testOauthRedirectPage.html" will render on your screen like this:

Pay special attention to the values for access_token and communityId.  The next step in this tutorial will use those values.  You will need to include the access token in all of your REST queries as the value of the "Authorization" HTTP header.
