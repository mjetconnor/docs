Post an Idea

There is more involved with posting ideas than posting comments or votes.  We will start with the simplest case and then delve into some variations.

Let's post a new idea to an inquiry.  As with the previous POST examples, the URL of the query will be the identifier of the thing we are posting to, in this case the inquiry's URI.  The body of the POST request describes what we want to post.  The type property with value "Idea" indicates that we want to post an idea, as opposed to a comment or vote or whatnot.  The title and description gives those initial values we want for our  new idea.

POST https://www.example.mindjet.com/api/ds1/q-1

with the usual headers, and this request body:

{

   type: "Idea",

   title: "Buy low, sell high",

   description: "This is a sure-fire way to make lots of money!"

}

The response contains the data for the newly created idea.

{

 "version": "1.0",

 "query": "POST https://www.example.mindjet.com/api/ds1/q-1",

 "xmlns": {

   "sief": "http://purl.org/sief/2013/10/",

   "_ds": "https://www.example.mindjet.com/api/ds1/",

   "_host": "https://www.example.mindjet.com/api/"

 },

 "results": [

   {

     "eid": "_ds:node-89523",

     "type": "Idea",

     "creator": {

       "eid": "_ds:user-523",

       "type": "User"

     },

     "created": "2013-12-01T09:41:52.000Z",

     "status": "open",

     "title": "Buy low, sell high",

     "description": "This is a sure-fire way to make lots of money!",

     "stage": {

       "eid": "_ds:j-1-0",

       "type": "Stage"

     },

     "inquiry": {

       "eid": "_ds:q-1",

       "type": "SuggestionBox"

     }

   }

 ]

}

 

This example will not work with all SpigitEngage inquiries because there may be additional required properties in the post body.  In particular, a SpigitEngage community or challenge may be configured with an "idea template" that defines certain extension properties as being required for ideas in that space.  Likewise, the community or challenge may have been configured with "categories".  If so, the posted idea must identify which category it belongs to.
