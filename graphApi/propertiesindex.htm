Properties Index

Alphabetical listing of all properties that appear in MindjetGraph.

Properties have a consistent semantic definition and usually the same basic behavior irrespective of the type of object or entity they appear.  Some properties only make sense in certain contexts.  Others are broadly applicable.  Most (but not all!) impose some kind of restriction on their allowed values.  In some cases, the type of allowed values depends on the context, specifically what kind of entity or object it is a property of.

Plural of Name

For a discussion of the impact of singular vs. plural property name usage, see:

Concept Overviews : Collections and Plural Property Names

REST API : Plural and Singular Property Names

Domain

The type of objects where the property may appear.  Most properties are not constrained in this way.  Their domain is shown in the table below as blank, indicating a domain of "Thing".

If you didn't know the type of a thing, and it has a property that is defined to have a certain domain, you can infer that the thing has that type.  For example, if you have an object, and that object has a value for a "replyTo" property, you know that that object must be a Comment, or a subclass of Comment.

Range

The range of a property is the datatype that all values of the property must belong to, either directly or as a subclass.  A blank value indicates a range of "Thing", meaning that any type of value is legal.

There may be additional constraints on a property's value when it appears in a certain context, such as being the property of a certain kind of entity.

Specific Domain,
Range in Specific Domain

For some properties, the range of allowable values can be more proscribed depending on a certain sub-domain.  For example, the action property may be broadly applied to any type of object, and in that broader context its value could likewise be any type of thing.  But when the action property appears in the context of a FusionTask object, its value must be a FusionAction.
