Property Expressions

Entries in the fields list can be property expressions (PE).  The simplest PEs are just property names as we have already seen.  In addition, a PE may contain an optional field list of its own, after the property name and a colon and surrounded by square brackets.

propertyName : [ nestedPE1, nestedPE2, nestedPE3, ...]

This list identifies the properties of the nested entity that should be included in the response.  Remember that by default, nested entities are returned in summary form.  This is how you can get different properties returned for nested elements.

Examples:

...?fields=title,creator

...?fields=title,creator:[name,mbox,phone]

In the first example, the response will include the "eid", "type", "title" and "creator" properties of the result entity.  For the creator, let's assume it's a User entity, the "eid", "type" and "name" properties are included in the response.

In the second example, the same property values are included for the top-level entity -  "eid", "type", "title" and "creator" .  But this time, the value for the creator property will be comprised of the "eid", "type", "name", "mbox" and "phone" properties.

A PE may also have an optional page size phrase, which is an asterisk followed by a number.

propertyName * 5

This specifies the number of elements to return, assuming that the value of the named property is a collection.

Examples:

...?fields=title,ideaScores*20,userScores*30

 

These phrases may be combined.  Any one property expression is made up of a property name, an optional fields phrase, and an optional page size phrase.

These elements may also be nested.  A property expression in a field list may contain its own field list, which contains other property expressions that have their own field lists.  And so on.

Example:

...?fields=title,description,ideas:[title,creator:[name,givenName],discussion:[comments*2]]*50

This may be easier to visualize written this way:

fields=

title,

description,

ideas:[

title,

creator:[

name,

givenName

],

discussion:[comments*2]

]*50

This request is asking for the response to include the "title", "description" and "ideas" properties of the top-level result entity.  As always, the "eid" and "type" properties are also included.  The value for the "ideas" property will use a page size of 50.  For each entry in that collection, the "title", "creator" and "discussion" (and "eid" and "type") properties are returned.  For each idea's creator, their "name" and "givenName"  (and "eid" and "type") properties are returned.  And lastly, for each idea's discussion, the first two comments are returned.
