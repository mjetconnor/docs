Get a Leaderboard

SpigitEngage captures a variety of metrics about things and people in order to measure how well they are doing.  These metrics depend on context.  For instance, a user's reputation or points balance may depend on which community you are interested in.  The context-dependent metrics for users are captured in UserScore entities.  The context-dependent metrics for ideas are captured in IdeaScore entities.

The data behind a leaderboard is typically a collection of scores of the appropriate type (user or idea), within a given context (a particular community or inquiry), and sorted by a particular property in the scores (reputation, points, approval-rating, activity-level).

The collection of scores for a context are usually easily available as properties in the entities that represent those contexts.  For instance, to get all the user scores whose context is a particular inquiry, use the "userScores" property of the inquiry.  Likewise, to get all of the idea scores whose context is an inquiry, use the "ideaScores" property of the inquiry.  In a similar manner, all of the scores for a given user, across all the various contexts, are available in the "scores" property on User entities.

Let's take the inquiry that we have seen earlier in this tutorial -- https://www.example.mindjet.com/api/ds1/q-1.  You can get the idea scores for that inquiry, in other words the idea scores whose context is that inquiry, like this:

The default page size for returned collections is 20 elements.  Maybe our hypothetical leaderboard widget displays just the top five, so we only need to get the first five entries:

GET https://www.example.mindjet.com/api/ds1/q-1/ideaScores?fields=approvalRating&sort=approvalRating;desc&pageSize=5

 

Lastly, maybe our leaderboard widget needs to display each idea's title, and the name and email address of the idea's creator.  We can get all of that in a single query by expanding on the fields query parameter.

GET https://www.example.mindjet.com/api/ds1/q-1/ideaScores?sort=approvalRating;desc&pageSize=20&fields=approvalRating,base:[title,creator:[name,mbox]]

This request asks for the approvalRating and base properties of each of the idea scores in the response.  The base of an idea score is the idea being scored.  So for those ideas we also request the title and creator properties.  And while we're at it, we also ask for the display name and email address for each creator.  This is now the response:
