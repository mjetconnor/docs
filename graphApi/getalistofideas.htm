Get a List of Ideas

Let's walk a little deeper into the graph by looking at the ideas associated with one of the inquiries.  We know from the description of the Inquiry class, and also from the more general Inquiry Concepts discussion, that inquiries have an ideaSet property.  And IdeaSets have an ideas property that is the collection of all Ideas participating in that inquiry.

Let's pick the first inquiry in the community that we have been looking at so far - "_ds:q-1".  As it happens, that one is a SuggestionBox, but that does not matter for what we are about to do.  All inquiries regardless of their specific type have ideaSets.

Let's get the ideaSet property of q-1 by using q-1's URI with the property name in an added path segment:

GET https://www.example.mindjet.com/api/ds1/q-1/ideaSet

It so happens that the value of this inquiry's ideaSet property is another entity, one whose eid is "_ds:iset-1" and has type IdeaSet.  (We knew that already from Inquiry Class.)  It has a bunch of properties.  One of them is the ideas property, which apparently is a collection that has 149 elements.  We can see the first three elements in this response, all of them Ideas.

Just to play a little bit with the query forms we have already seen, we could have gotten just the ideas in this inquiry in a couple of ways.  Let's start with:

GET https://www.example.mindjet.com/api/ds1/q-1/ideaSet/ideas

This form starts with the inquiry's URI -- the part up to ".../ds1/q-1"  -- then appends "/ideaSet" to select that property from the inquiry, and then appends "/ideas" to select that property from the resulting ideaSet.  See how the path segments can be cascaded?  Again, this is just like using a dot operator in a scripting language.

Or, since we now know the entity ID of the ideaSet ("_ds:iset-1"), we can query for it directly, but still appending the "/ideas" segment:

GET https://www.example.mindjet.com/api/ds1/iset-1/ideas

Really, the default response includes a lot of information that we cut away here for brevity.  You could make your request a bit more targetted by specifying exactly which properties you are interested in, using the fields query parameter:

GET https://www.example.mindjet.com/api/ds1/node-75494?fields=title,creator,created,status,stage,inquiry,subjects
