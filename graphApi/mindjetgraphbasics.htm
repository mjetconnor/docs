MindjetGraph Basics

Resources are the atoms of the Semantic Web.  A resource is any kind of identifiable thing.  It's the internet, so of course those identifiers are URIs.

Entities are what we call resources when they come out of MindjetGraph.

Properties are what we call the relationships between things in MindjetGraph.  The Mindjet Graph is made up of entities related to each other by their properties.

SIEF (Social Innovation Enterprise Framework) is the name of the MindjetGraph vocabulary.  It defines exactly what the various class and property names mean.  The class and property names described in this document are all defined in the context of the "http://purl.org/sief/2013/10/terms/" namespace.

Use the REST API to get the properties for an entity.  Since all MindjetGraph entity's have a URI identifier, an HTTP GET request using that URI will get the entity's data.

The MindjetGraph API uses OAuth 2.0 as its authentication scheme.
