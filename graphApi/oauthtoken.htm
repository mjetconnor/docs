/oauth/token

Execute an HTTP POST request to this URL:

authorizationServiceURL/oauth/token

The HTTP GET method is not accepted.  POST is required for this query.  The request parameters are included in the body of the POST request.  The only format currently supported is "application/x-www-form-urlencoded".

It is strongly suggested, and required by most authorization service endpoints, that the request is made using the "https://" protocol.  This makes the request secure by encrypting the parameters in the request body.

The request body must include a value for the grant_type parameter.  The value of that parameter determines which other parameters are expected.  The value must be one of these: authorization_code, refresh_token, or password.


Request Parameter

Description

grant_type

"authorization_code"

code

An authorization code that was previously returned from a request to

/oauth/authorize?response_type=code

client_id

The key of the application that is making this call.

client_secret

The secret of the application.

redirect_uri

Even though the response to this request will be returned directly and not by way of a 302 redirect, the redirect URL is still required here.  This provides an additional  check against unwanted accesses.


Request Parameter

Description

grant_type

"refresh_token"

refresh_token

A refresh token value that was received in an earlier access token response.

client_id

The key of the application that is making this call.

client_secret

The secret of the application.


Request Parameter

Description

grant_type

"password"

client_id

The key of the application that is making this call.

client_secret

The key of the application that is making this call.

username

User authentication will be performed using this username.

password

User authentication will be performed using this password.

 

In all three cases, the body of the response to the request contains an access token response.  Because these responses are returned directly back to the request, there is no need for a state parameter to coordinate responses with requests.
