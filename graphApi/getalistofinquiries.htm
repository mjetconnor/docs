Get a List of Inquiries

Take another look at the response we got back two steps ago when we asked for the "_ds:comm-1" entity.  One of the properties was the inquiries property:

 

     ... etc. ...

     "inquiries": {

       "data": [

         {

           "eid": "_ds:q-1",

           "type": "SuggestionBox"

         },

         {

           "eid": "_ds:q-816",

           "type": "Challenge"

         },

         {

           "eid": "_ds:q-819",

           "type": "Challenge"

         }

       ],

       "paging": {

         "next": "https://www.example.mindjet.com/api/ds1/comm-1/inquiries?after=2"

       },

       "totalSize": 7

     },

     ... etc. ...

This is an example of a value that is a collection.  The response format of collections is described in detail in Collection Result Elements.

By default for this kind of response, only the first three elements are returned.  To get more, execute another GET request using the URL provided in the paging.next element:

GET https://www.example.mindjet.com/api/ds1/comm-1/inquiries?after=2

You can specify a certain block size by appending the pageSize query parameter:

GET https://www.example.mindjet.com/api/ds1/comm-1/inquiries?after=2&pageSize=100

will return the next 100 elements in the collection.  (That is plenty for this example, since we can see that this inquiries collection only has 7 elements in it.)

Notice that the elements in this collection are all entities themselves.  This should start to look familiar -- they each have an eid and type property.  We can see right off that the first entry is a kind of inquiry called a SuggestionBox, and the next two are Challenges.  We can "walk the graph" to their detail data in the familiar way:

GET https://www.example.mindjet.com/api/ds1/q-1

GET https://www.example.mindjet.com/api/ds1/q-816

GET https://www.example.mindjet.com/api/ds1/q-819

We could send a request that specifically asks for the inquiries property of the community without getting all of the other properties that the community might have.  There are two ways to do this.  One is to use the fields query parameter:

GET https://www.example.mindjet.com/api/ds1/comm-1?fields=inquiries

This will return only the inquiries property for comm-1.  Well, that's a little fib.  You always get the eid and type properties in responses for entities, so what you will get back is the eid, type and inquiries properties for comm-1.

{

 "version": "1.0",

 "query": "GET https://www.example.mindjet.com/api/ds1/comm-1",

 "xmlns": {

   "sief": "http://purl.org/sief/2013/10/",

   "_ds": "https://www.example.mindjet.com/api/ds1/",

   "_host": "https://www.example.mindjet.com/api/"

 },

 "results": [

   {

     "eid": "_ds:comm-1",

     "type": "Community",

     "inquiries": {

       "data": [

         {

           "eid": "_ds:q-1",

           "type": "SuggestionBox"

         },

         {

           "eid": "_ds:q-816",

           "type": "Challenge"

         },

         {

           "eid": "_ds:q-819",

           "type": "Challenge"

         }

       ],

       "paging": {

         "next": "https://www.example.mindjet.com/api/ds1/comm-1/inquiries?after=2"

       },

       "totalSize": 7

     }

  }

 ]

}

The other way is to add a path segment to the query URI that uses the property name:

GET https://www.example.mindjet.com/api/ds1/comm-1/inquiries

In this case, the response is, directly, the value of the inquiries property, as opposed to being the community entity limited to just that property.  Using the additional path segment is analogous to doing a dot op in a scripting language.

{

 "version": "1.0",

 "query": "GET https://www.example.mindjet.com/api/ds1/comm-1/inquiries",

 "xmlns": {

   "sief": "http://purl.org/sief/2013/10/",

   "_ds": "https://www.example.mindjet.com/api/ds1/",

   "_host": "https://www.example.mindjet.com/api/"

 },

 "results": [

   {

     "data": [

       {

         "eid": "_ds:q-1",

         "type": "SuggestionBox"

       },

       {

         "eid": "_ds:q-816",

         "type": "Challenge"

       },

       {

         "eid": "_ds:q-819",

         "type": "Challenge"

       },

       {

         "eid": "_ds:q-560",

         "type": "Challenge"

       },

       {

         "eid": "_ds:q-571",

         "type": "Challenge"

       },

       {

         "eid": "_ds:q-516",

         "type": "Challenge"

       },

       {

         "eid": "_ds:q-508",

         "type": "Challenge"

       }

     ]

  }

 ]

}

 

Also notice that when the "top level" response is a collection, the default page size is bigger.  (The default size is 20.)  We got all seven elements in the response, not just the first three and a "next" link.
