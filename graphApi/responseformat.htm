The specific format of the query response is the one requested by either the Content-Type header or format query parameter.  The default is JSON.

Regardless of format, or even the kind of query or HTTP method, all responses have a common structure.  The top level elements are always:

Top-level Response Element

Required

Description

version

MUST

The version of the MindjetGraph REST response syntax.  Currently always "1.0".

query

SHOULD

Echoes the HTTP method and URL of the query to which this response was made.

xmlns

SHOULD

If any XML namespace prefixes were used in formatting this response, their mappings are given here.  Definitions for "_ds" and "_host" are typically included.

results

MAY

The array of results, if any, in response to the query.

error

MAY

An error message.  A response typically includes either a results element or an error element but not both.

The response to most GET queries is a results element that contains exactly one entry - the result of the query.  If the response to the query is a collection value, say a list of ideas or users, the results element still contains exactly one entry, but that one entry will contain a collection value.

Example response in JSON:

{

    "version": "1.0",

    "query": "GET https://www.example.mindjet.com/api/ds1/tenant-1",

    "xmlns": {

        "_ds": "https://www.example.mindjet.com/api/ds1/",

        "_host": "https://www.example.mindjet.com/api/",

        "_sief": "http://purl.org/sief/2013/10/"

    },

    "results": [

        {

            "eid": "_ds:tenant-1",

            "type": "Tenant",

            "created": "2013-03-12T17:15:21.000Z",


            ... etc. ...

        }

    ]

}
