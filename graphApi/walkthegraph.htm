Walk the Graph

Now that we have a tentative first step into The Graph, let's see what we find if we take another step.

In the response to our query for "comm-1", one of the many properties is a value for the creator property.  This identifies who created the community.

... etc. ...

     "creator": {

       "eid": "_ds:user-59",

       "type": "User"

     },

     ... etc. ...

We can tell from this that the value of the creator property is another entity, because it is something that has its own eid property.  That entity's ID, after expanding the "_ds:" prefix, is "https://www.example.mindjet.com/api/ds1//user-59".

We can also see that that entity is a User, as identified by its type property.

That's all we know about the community's creator from this response.  To find out more, we have to do another GET request using this new URI:

GET https://www.example.mindjet.com/api/ds1/user-59

(Did you remember to use the "Authorization" header?)

The following shows what you might get back in response.  This is again somewhat abbreviated.  There could be a number of additional properties included in your response, depending on the details of your SpigitEngage version.

{

 "version": "1.0",

 "query": "GET https://www.example.mindjet.com/api/ds1/user-59",

 "xmlns": {

   "sief": "http://purl.org/sief/2013/10/",

   "_ds": "https://www.example.mindjet.com/api/ds1/",

   "_host": "https://www.example.mindjet.com/api/"

 },

 "results": [

   {

     "eid": "_ds:user-59",

     "type": "User",

     "name": "Jonathan Dough"

     "familyName": "Dough",

     "givenName": "Jonathan",

     "img": {

       "type": "Image",

       "url": "https://www.example.mindjet.com/resources/users/59/avatar.jpg"

     },

     "creator": {

       "eid": "_ds:user-1",

       "type": "User"

     },

     "created": "2013-03-12T17:15:45.000Z",

     "mbox": "jdoe@mindjet.com",

     "status": "open",

     "userAccount": {

       "eid": "_ds:ua-59",

       "type": "UserAccount"

     }

   }

 ]

}

 

This response contains references to two more entities in the values of the creator and userAccount properties.  You can take more steps in the graph by requesting the data for those entities as well:

GET https://www.example.mindjet.com/api/ds1/user-1

GET https://www.example.mindjet.com/api/ds1/ua-59

While we're here, take a look at the value for the img property in this example.  It looks a bit like a MindjetGraph entity - it's an object value surrounded by braces "{" ... "}", and it even has a property that looks like a URI.  But there is no eid property, so this is not an entity.
