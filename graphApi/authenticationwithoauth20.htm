MindjetGraph REST queries are authenticated using the OAuth 2.0 regime.  All queries must be tagged with an "Authorization" HTTP header whose value is "Bearer accessToken".

That part is pretty easy.  The fun is in getting that access token.

The MindjetGraph REST OAuth 2.0 implementation religiously follows the OAuth 2.0 v20 specification.  If you check out that document, you may want to keep in mind this terminology:

Query Parameter

Description

client

This is what OAuth 2.0 calls your application.  This is not the user.

resource owner

An end-user, acting in tandem with a MindjetGraph authorization endpoint.  In these Help Docs, we call this the authorizationServiceURL

resource server

For us, this will be a gatewayURI, as discussed in Rest API Entity IDs.  This is implicit in each MindjetGraph entity's identifying URI.
