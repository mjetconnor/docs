Post a Comment to an Idea

Now let's post a comment to that idea.  The query's URL will be the idea's URI, just like for the GET queries, but this time we will use the HTTP POST method.

The body of the POST request describes what we want to add to the idea.  Since we want to make a new comment, we have to give the values we want to use to create that comment.

POST https://www.example.mindjet.com/api/ds1/node-75494

with headers:

Content-Type: application/json

Authorization: Bearer 914a9b4d83fa7333d3b517643a30bac2

and request body:

{

   type:"Comment",

   content:"I love this idea.  It's awesome!",

}

or the same thing using form encoding:

Content-Type: application/x-www-form-urlencoded

Authorization: Bearer 914a9b4d83fa7333d3b517643a30bac2

and request body:

type=Comment&content=I+love+this+idea%2E++It%27s+awesome%21

The response to the request will be the data for the newly created comment, in exactly the same format that we have seen so far.

{

 "version": "1.0",

 "query": "POST https://www.example.mindjet.com/api/ds1/node-75494",

 "xmlns": {

   "sief": "http://purl.org/sief/2013/10/",

   "_ds": "https://www.example.mindjet.com/api/ds1/",

   "_host": "https://www.example.mindjet.com/api/"

 },

 "results": [

   {

     "eid": "_ds:node-91024",

     "type": "Comment",

     "creator": {

       "eid": "_ds:user-523",

       "type": "User"

     },

     "created": "2013-12-01T13:13:52.000Z",

     "content": "I love this idea.  It's awesome!",

     "forum": {

       "eid": "_ds:d-75494",

       "type": "Discussion"

     },

     "status": "open"

   }

 ]

}
