Matrix Parameters

We can extend queries a little bit more by adding matrix parameters to a property segment in the query URL, using this syntax:

entityURI/propertyName;selectionExpr

The value of the property named by propertyName is assumed to be a collection of some sort.  The part of the path segment after the semicolon (";") is a rule for selecting a subset of elements from that collection.  This simplest kind of rule is an equals comparison, expressed this way:

entityURI/propertyName;pname=value

Read this as follows:  Get the value of propertyName from the entityURI entity.  It will be a collection.  For each element in that collection, find the elements that have a pname property, and whose value for that property matches the requested value.  The result of the query is a collection whose elements are all the matching elements.

Collection entries that do not have a property value for pname will not match, and therefore will not be included in the response.

The other comparison operators also work:

entityURI/propertyName;pname>value

entityURI/propertyName;pname<value

entityURI/propertyName;pname>=value

entityURI/propertyName;pname<=value

There may be multiple selection expressions, aka matrix parameters, in the same property segment.  The more complete syntax is:

entityURI/propertyName;selectionExpr;selectionExpr;...

The selection expressions are effectively AND'd together to create a single selection criterion.  For example:

http://example.mindjet.com/api/ds1/comm-1/inquiries;type=Challenge;title>=c;title<d

where

entityURI: http://example.mindjet.com/api/ds1/comm-1
propertyName: inquiries
matrix parameters: type=Challenge (AND) title>=c (AND) title<d

This will return all of the inquiries in comm-1 that are challenges and have a title in the range of "c".."d" (case insensitive).

These matrix parameters are part of the property segment.  The whole thing can be repeated again, with yet another property segment following a matrix parameter, and that property segment may include matrix parameters of its own.

Remember that normal URL encoding rules apply.  Be sure to use percent encoding where necessary in the value expression.  Also be sure to avoid whitespace like blank characters, unless of course they are encoded as %20.
