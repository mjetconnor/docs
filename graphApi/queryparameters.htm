Query Parameters

These query parameters are supported:

Query Parameter

Description

format

Values may be "json" or "xml".  The response will be formatted according to this value.

If both this query parameter and the Content-Type header are specified, this query parameter takes precedence.

If neither this parameter nor the Content-Type header are specified, the default is "json".

fields

When entity data is returned in a response, certain properties are included by default, as described in Entity Result Elements.  The fields query parameter allows you to tailor the response to include exactly the element data you need. At its simplest, a fields value is a comma-delimited list of property names.  The response will include just those properties named in the list. Much more intricate denormalization patterns are also available. A full description of this value is at Fields Query Parameter.

pageSize

When the result is a collection, this specifies the page size of the result.  For collections that have more elements than this number, only the first elements are returned up to this number.  A "next" URL will be included in the collection value that may be used to retrieve subsequent blocks of elements from the collection.

If not specified, the default is "20".

sort

If specified, and if the result is a collection, this specifies the sort order of the elements returned for that collection.  Paging still applies, as described above for the pageSize parameter.

The value of this parameter is a property name, optionally followed by ";desc" or ";asc" to specify descending or ascending order.  For example: ...?sort=created;desc.  The default direction is ascending.

If not specified, collections are returned in their natural order.


Example:

GET https://www.example.mindjet.com/api/ds1/tenant-1/communities?sort=created;desc&pageSize=100&format=json

(Please ignore any line-folding in this example.  There can be no whitespace in URLs.)
